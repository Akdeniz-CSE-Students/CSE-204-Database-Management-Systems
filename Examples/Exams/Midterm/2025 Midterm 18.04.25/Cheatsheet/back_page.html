E-R Modeling

Basic Elements

  • Entity: Object or concept (rectangle)
  • Attribute: Property of entity (oval)
  • Relationship: Association between entities (diamond)

Attribute Types

  • Simple: Atomic, indivisible
  • Composite: Can be divided into smaller parts
  • Single-valued: One value per entity
  • Multi-valued: Multiple values per entity
  • Derived: Calculated from other attributes

Cardinality Constraints

  • One-to-One (1:1): Each entity in A is related to at most one entity in B
  • One-to-Many (1:*): Each entity in A is related to any number of entities in B
  • Many-to-Many (*:*): Entities in A and B can be related to any number of entities

Participation Constraints

  • Total: Every entity must participate in relationship (double line)
  • Partial: Some entities may not participate (single line)

Converting E-R to Relations

  1. Create table for each entity with primary key
  2. Create table for each M:N relationship
  3. For 1:N, place foreign key in N-side table
  4. For 1:1, place foreign key in either table (preferably with total participation)

DB Analysis

Database Design Process

  1. Requirements collection and analysis
  2. Conceptual database design (E-R model)
  3. Logical database design (relational schema)
  4. Schema refinement (normalization)
  5. Physical database design (storage structures)
  6. Security design

Requirements Analysis

  • Identify entities and relationships
  • Determine data constraints
  • Define user operations (queries, updates)
  • Document requirements

Schema Design Principles

  • Avoid redundancy
  • Avoid null values when possible
  • Ensure entity integrity
  • Ensure referential integrity
  • Consider query performance

Common Design Pitfalls

  • Insufficient normalization (redundancy)
  • Over-normalization (performance issues)
  • Poor primary key selection
  • Inadequate foreign key constraints
  • Ignoring data integrity rules

Advanced SQL

Subqueries

SELECT column FROM table WHERE column IN (SELECT column FROM table);

Aggregation Functions

  • COUNT(): Number of rows
  • SUM(): Sum of values
  • AVG(): Average of values
  • MIN(): Minimum value
  • MAX(): Maximum value

GROUP BY with HAVING

SELECT column, COUNT(*) FROM table GROUP BY column HAVING COUNT(*) > value;

ALTER TABLE

ALTER TABLE table ADD column datatype; ALTER TABLE table DROP COLUMN column; ALTER TABLE table MODIFY COLUMN column datatype;

Views

CREATE VIEW view_name AS SELECT columns FROM tables WHERE condition;
