Midterm Examination Cheatsheet

Relational Model

Relations and Relationships

Relation: A table in a database; similar to a table

Relationship: Connection between tables with common attributes

Entity Types

  • Strong Entity: Does not depend on another entity for existence
  • Weak Entity: Depends on another entity for existence

Keys

  • Primary Key: Uniquely identifies each row; cannot be null
  • Foreign Key: References primary key in another table
  • Candidate Key: Potential primary key; uniquely identifies rows
  • Super Key: Any set of attributes that uniquely identifies a row
  • Alternate Key: Candidate key not selected as primary key

Integrity Constraints

  • Entity Integrity: Primary key cannot be null
  • Referential Integrity: Foreign key must match existing primary key or be null
  • Domain Integrity: Values must conform to defined data types

Relationship Traps

  • Fan Trap: One entity has many relationships with two others, but relationship between these others is necessary
  • Chasm Trap: Participation with entity not required, but finding counterpart with third entity is necessary

SQL

Creating Tables

CREATE TABLE TableName ( column1 datatype constraints, column2 datatype constraints, ... PRIMARY KEY (column), FOREIGN KEY (column) REFERENCES OtherTable(column) );

SQL Constraints

  • PRIMARY KEY - Uniquely identifies each record
  • FOREIGN KEY - Links data between tables
  • NOT NULL - Column cannot have NULL value
  • UNIQUE - All values in column must be different
  • DEFAULT - Sets default value for column
  • CHECK - Ensures values meet specified condition

SELECT Queries

SELECT column1, column2, ... FROM table1, table2, ... WHERE condition GROUP BY column HAVING condition ORDER BY column ASC/DESC;

JOIN Operations

SELECT columns FROM table1 INNER JOIN table2 ON table1.column = table2.column; SELECT columns FROM table1 LEFT JOIN table2 ON table1.column = table2.column;

Data Manipulation

INSERT INTO table (column1, column2, ...) VALUES (value1, value2, ...); UPDATE table SET column1 = value1, column2 = value2, ... WHERE condition; DELETE FROM table WHERE condition;

Normalization

Functional Dependencies

Notation: X → Y means X determines Y

  • Functional Dependency: Attribute(s) determines value of another attribute(s)
  • Full Functional Dependency: X→Y where Y depends on entire X, not subset
  • Transitive Dependency: X→Y, Y→Z creates X→Z
  • Partial Dependency: Part of composite key determines non-key attribute

Normal Forms

  • 1NF: All attributes are atomic (no repeating groups)
  • 2NF: In 1NF and no partial dependencies on primary key
  • 3NF: In 2NF and no transitive dependencies
  • BCNF: For X→Y, X must be a superkey

Normalization Process

  1. Identify all functional dependencies
  2. Find candidate keys
  3. Check for partial dependencies (violate 2NF)
  4. Check for transitive dependencies (violate 3NF)
  5. Decompose relations to achieve desired normal form
Remember: Normalization reduces redundancy but may impact performance due to increased JOINs
